Descripción detallada

Permite jugar al buscaminas en un tablero fijo, pero variable en el programa, de 8x8.

Datos de entrada: nombres de archivos: cadenas de caracteres; opcion de menu: caracter; valores de celdas: enteros positivos pequenos.

Datos de salida: tablero: caracteres, enteros positivos pequenos; menu, mensajes finales: cadenas de caracteres.

◆ AbreCelda()

void AbreCelda ( Tablero tablero,
unsigned short fil,
unsigned short col )

Modifica el estado de posiciones de un array tablero.

Explora recursivamente las celdas vecinas si estas no estan en contacto con posiciones que contengan minas. Utiliza una logica similar al algoritmo flood-fill en 8 direcciones.

Precondición
El tablero debe tener sus valores inicializados; tomar especial atencion con valores enteros.
Parámetros
[in,out]tableroArray del que leer y modificar estados de celda(s).
[in]filNumero de fila para la celda desde la que explorar.
[in]colNumero de columna para la celda desde la que explorar.

◆ InicializaDesdeFichero()

void InicializaDesdeFichero ( Tablero tablero,
ifstream & f )

Lee datos de un archivo abierto con formato fijo y los escribe en un array de tablero.

Las posiciones del archivo que no estan afectadas se inicializan con valores por defecto, contrarios a los de las celdas-minas.

Precondición
El archivo recibido debe estar abierto; se asume la ausencia de badbit, failbit y eofbit.
El archivo debe contener posiciones validas, y un maximo de valores ajustado al maximo de minas establecido; no se realizara ninguna comprobacion.
Parámetros
[out]tableroArray de filas y columnas a inicializar.
[in]fArchivo de entrada del que leer la informacion.

void LeeJugadoresFichero ( VectorJ jugadores,
unsigned short & tam,
ifstream & f )

Lee jugadores de un archivo de entrada de formato fijo, y almacena los datos encontrados en un array de jugadores.

Precondición
El archivo recibido debe estar abierto; se asume que no hay ningun badbit, failbit o eofbit.
El archivo debe contener un numero de jugadores en el rango 0-100 para evitar que el vector no recoja todos los jugadores. Los datos de los jugadores tambien deben ser conformes con los datos del archivo de ejemplo; no se realizara ninguna comprobacion.
Parámetros
[out]jugadoresArray de jugadores en el que escribir datos.
[out]tamTamano del array rellenado con los jugadores del archivo.
[in]fArchivo de entrada con jugadores existentes.
